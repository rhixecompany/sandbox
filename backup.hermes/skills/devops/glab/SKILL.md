## Description

Expert guidance for GitLab CLI (glab) to manage GitLab resources from terminal. Covers merge requests, issues, pipelines, and repositories.

## When to Use

- Creating or reviewing merge requests
- Managing GitLab issues
- Monitoring or triggering CI/CD pipelines
- Cloning or forking repositories
- Automating GitLab workflows
- Troubleshooting authentication

## When NOT to Use

- GitHub (use GitHub CLI instead)
- Web-based GitLab interface
- Non-GitLab repositories
- Local git operations

## Workflow

### Phase 1: Setup & Authentication

- Install glab CLI
- Configure authentication
- Verify access to repositories

### Phase 2: Manage Resources

- Create or list issues
- Create or review merge requests
- Link issues to MRs
- Manage labels and milestones

### Phase 3: Monitor Pipelines

- Check pipeline status
- Trigger pipeline runs
- View job logs
- Debug failures

### Phase 4: Automate Workflows

- Create scripts for common tasks
- Set up CI/CD integration
- Automate repetitive operations
