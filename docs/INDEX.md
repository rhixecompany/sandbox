## 📌 Installation Plan

Below is the list of MCP servers that can be installed via `hermes mcp add`.  
Replace the placeholder values with the actual API keys or authentication tokens that you will supply (commonly those are environment variables).  

| MCP Server | `hermes mcp add` Command | Notes |
|------------|--------------------------|------|
| **Context7** | `hermes mcp add @context7/mcp` | Requires a Context7 API key – set `CONTEXT7_API_KEY`.
| **Google Maps** | `hermes mcp add @cablate/mcp-google-map` | Use `GOOGLE_MAPS_API_KEY` (or supply via `--api-key`).
| **Markdownify** | `hermes mcp add @zcaceres/markdownify-mcp` | No authentication – runs locally.
| **MarkItDown** | `hermes mcp add @KorigamiK/markitdown-mcp` | No external API.
| **Django MCP Server** | `hermes mcp add @omarbenhamid/django-mcp-server` | No external API.
| **GitHub Official** | `hermes mcp add @github/github-mcp-server` | Requires a GitHub personal access token – pass `GITHUB_TOKEN`.
| **Redis** | `hermes mcp add @redis/redis-mcp` | Use `REDIS_URL` if you need authentication.

> **How to run the install**:
> ```bash
> hermes mcp add <server>
> ````
> The CLI will prompt you for any missing auth values.  

After installing, verify a few picks by running:
> ```bash
> hermes mcp inspect <server>
> ```
> to list the exposed tools and test them with `hermes run "<tool> <args>"`.
