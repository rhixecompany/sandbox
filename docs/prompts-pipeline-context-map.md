# Prompts Pipeline Context Map

## Scope

Apply the following pipeline to prompt assets:

1. Convert plaintext prompt sources from `Prompts/*.txt`.
2. Refine and validate `Prompts/*.md` using context-map, boost-prompt, safety review, implementation-plan update, and prompt-builder rules.

## Risk Assessment

- Converting terse plaintext prompts can accidentally introduce assumptions not in source.
- Legacy prompts may contain Windows paths and destructive actions that need explicit safeguards.
- Inconsistent frontmatter fields across prompts can reduce downstream tooling compatibility.
- No automated tests exist for prompt quality, so regressions must be manually reviewed.
