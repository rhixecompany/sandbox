## Architectural Patterns

### App Router Full-Stack Pattern
- Server Components for data fetching and SEO
- Client Components for interactive library features
- API route handlers for external integrations
- Server Actions for book management operations

### Serverless Database Pattern
- Neon (serverless PostgreSQL) for scalable database
- Drizzle ORM for type-safe queries
- Connection pooling for serverless environment
- Migration via `drizzle-kit`

### Redis Caching Pattern
- Upstash Redis for session caching
- Rate limiting for API endpoints
- Cache frequently accessed library data

## Architecture Decisions

1. **Drizzle ORM** — Type-safe, lightweight SQL ORM suited for serverless environments
2. **Neon Serverless PostgreSQL** — Scalable, autoscaling PostgreSQL for serverless Next.js
3. **Upstash Redis** — Serverless Redis for caching, rate limiting, and session management
4. **NextAuth.js** — Flexible authentication with multiple provider support
5. **Server Actions** — Simplified form handling and data mutations
