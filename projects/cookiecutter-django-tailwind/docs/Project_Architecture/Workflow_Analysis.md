## Workflows

### Workflow 1: Project Scaffolding via Cookiecutter

**Description:** User generates a new Django project from the template using Cookiecutter.

**Entry Points:**
- CLI: `cookiecutter gh:your-org/cookiecutter-django-tailwind`

**Steps:**
1. User runs `cookiecutter` with the template URL
2. Cookiecutter prompts for project variables (project name, directory, etc.)
3. Cookiecutter copies template files with rendered variables
4. Resulting project inherits the entire template structure:
   - Django project with settings (`base.py`, `local.py`, `production.py`)
   - Tailwind CSS configuration
   - Docker Compose files for development and production
   - Pre-configured testing (pytest, coverage)
   - Pre-commit hooks (Black, ruff, mypy, djlint)
5. User follows setup instructions: venv, pip install, migrate, createsuperuser

**Error Handling:**
- Cookiecutter not installed → user prompted to `pip install cookiecutter`
- Template fetch failure → check network/GitHub access
- Duplicate directory → Cookiecutter prompts before overwriting

**Test Patterns:**
- Generated project must pass `python manage.py check --deploy`
- Pytest coverage minimum threshold
- Pre-commit hooks validate generated code quality

### Workflow 2: Development Server with Tailwind

**Description:** Running the generated project in development with Tailwind CSS compilation.

**Entry Points:**
- `python manage.py runserver`
- `python manage.py tailwind start`

**Steps:**
1. User starts Django dev server (`python manage.py runserver`)
2. User starts Tailwind CSS watcher (`python manage.py tailwind start`)
3. Tailwind compiles CSS on file changes
4. Django serves the compiled CSS
5. User develops with hot-reload for both Python and CSS changes
6. Tests run via `pytest --cov`

**Error Handling:**
- Tailwind not installed → `python manage.py tailwind install`
- Migration conflicts → `python manage.py makemigrations --merge`
- Missing settings → check `base.py` → `local.py` override chain
