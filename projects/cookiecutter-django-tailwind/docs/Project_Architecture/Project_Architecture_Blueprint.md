## Architectural Patterns

### Cookiecutter Template Pattern
- Generates new projects from cookiecutter `{{ cookiecutter }}` variables
- Pre-configured with modern Django best practices
- Option selection via interactive prompts

### Environment-Specific Settings
Settings follow a layered inheritance:
1. `base.py` — Settings shared across all environments
2. `local.py` — Dev-specific settings (debug on, SQLite)
3. `production.py` — Production settings (debug off, PostgreSQL, Gunicorn)

### Optional Feature Toggle
- Django REST Framework (optional)
- Alpine.js / htmx (optional)
- Environment-specific requirements files

## Architecture Decisions

1. **Cookiecutter for scaffolding** — Generate consistent project structures
2. **django-environ for secrets** — Environment variables via `.env` files
3. **Tailwind utility-first** — Minimal custom CSS, utility classes throughout
4. **WhiteNoise for static** — Self-serve static files without external storage
5. **pytest for testing** — Modern testing framework over unittest
