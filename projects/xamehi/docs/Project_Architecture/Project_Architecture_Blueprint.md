## Architectural Patterns

### Dual-Backend Pattern
- **Django (Primary)**: Main API server, database management, authentication
- **Express (Secondary)**: Lightweight API server, real-time features, proxy
- Both backends share the same PostgreSQL database

### Service Port Mapping
- **Django**: Port 8000 (primary API)
- **Express**: Port 5000 (secondary services)
- **React**: Port 3000 (frontend dev server)

### React SPA Pattern
- Create React App for frontend
- Axios for API communication with both backends
- CORS configured for development and production

## Architecture Decisions

1. **Dual backend architecture** вҖ” Django for heavy data/crud operations, Express for light/real-time services
2. **Create React App** вҖ” Mature React toolchain for frontend development
3. **Port separation** вҖ” Each service on dedicated port for independent scaling
4. **Axios for API calls** вҖ” Promise-based HTTP client for both Django and Express endpoints
5. **Environment-based config** вҖ” `.env` files for secrets and configuration
