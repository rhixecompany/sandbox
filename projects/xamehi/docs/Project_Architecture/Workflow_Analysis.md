## Project Overview

**Project:** xamehi
**Type:** Legacy Dual-Backend + React Application
**Stack:** Django + DRF + Express (Node.js) + React 18 (CRA) + PostgreSQL
**Architecture Pattern:** Multi-Service (Django API + Express API + React SPA)
**Entry Point:** Three services: Django (`:8000`), Express (`:5000`), React (`:3000`)

A legacy full-stack application with two backend services (Django REST + Express) and a React 18 frontend (Create React App), with CORS proxying between them.

## Workflows

### Workflow 1: User Request Routing (Full-Stack Flow)

**Description:** User interacts with React frontend which proxies requests to Django or Express backends depending on the endpoint.

**Entry Points:**
- **Frontend:** React 18 (CRA) on port 3000 (`npm start`)
- **Django Backend:** `xamehi/urls.py` on port 8000
- **Express Backend:** `node index.js` or `npm run server` on port 5000

**Steps:**
1. User opens React app at `localhost:3000`
2. React renders UI components (from CRA template)
3. Axios requests are proxied to backends:
   - `/api/django/...` → Django on port 8000
   - `/api/express/...` → Express on port 5000
4. Django handles data operations via DRF (`xamehi/urls.py`)
5. Express handles specific service endpoints
6. Both backends query PostgreSQL for data
7. Responses flow back: DB → Backend → Axios → React → User

**Sequence Diagram:**
```

    |                        |                       |                       |
    |-- API request -------->|                       |                       |

    |                        |-- handle request ---->|                       |
    |                        |   (Express route)     |                       |
    |                        |                       |                       |
    |                        |-- query DB ---------->|                       |
    |                        |<-- query result ------|                       |
    |                        |                       |                       |
    |<-- JSON response ------|                       |                       |
    |                        |                       |                       |
    |-- API request -------->|                       |                       |
    |   /api/django/...      |                       |                       |
```

**Error Handling:**
- CORS misconfiguration → `django-cors-headers` + Express CORS middleware
- Backend unavailable → Axios error handling, fallback UI
- Port conflicts → documented in README
- Production HTTPS required for security

**Test Patterns:**
- Django tests: `python manage.py test`
- React tests: `npm test`
- Manual API testing

### Workflow 2: Development Environment Setup

**Description:** Running all three services locally for development.

**Entry Points:**
- `npm start` — React dev server (port 3000)
- `npm run server` — Express with Nodemon (port 5000)
- `python manage.py runserver` — Django dev server (port 8000)

**Steps:**
1. Start Django backend: `python manage.py migrate && python manage.py runserver`
2. Start Express backend: `npm run server` (Nodemon for auto-restart)
3. Start React frontend: `npm start` (CRA dev server)
4. Verify all three services are running
5. Navigate to `localhost:3000` for the app
6. Frontend proxies API calls to appropriate backend

**Error Handling:**
- Missing migrations → `python manage.py makemigrations` first
- Express port in use → kill process or change port
- `.env` missing → copy template, set SECRET_KEY, DB creds
