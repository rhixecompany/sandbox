## Architectural Patterns

### App Router Pattern
- Server Components by default for performance
- Client Components where interactivity required
- Route handlers for API endpoints
- Server Actions for mutations

### Database Access Layer
- Drizzle ORM for type-safe SQL
- Schema defined in `src/db/schema.ts`
- Migrations via `drizzle-kit`
- Drizzle Studio for DB management

### Financial Integration
- Plaid sandbox for development (token exchange, transaction sync)
- Dwolla for ACH transfers and user funding
- Webhooks for Plaid/Dwolla event handling

## Architecture Decisions

1. **App Router** — Leverages Next.js 16 App Router for server components, streaming, and edge capabilities
2. **Drizzle ORM** — Type-safe SQL with better performance than full ORMs
3. **Upstash Redis** — Serverless Redis for rate limiting and session caching
4. **Plaid + Dwolla** — Dual financial API integration for bank connectivity and transfers
5. **shadcn/ui** — Component library built on Radix UI primitives for accessibility

## Extensibility Points

- Add new banking features via App Router route groups
- Add new financial integrations in `src/lib/`
- Extend schema in `src/db/schema.ts`
- Add new UI components via shadcn CLI (`npx shadcn add`)
- Add new API endpoints as route handlers
