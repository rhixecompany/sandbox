## Architectural Patterns

### Single-Script CLI Pattern
Each script is independently executable:
- `main_noplaylist.py` — Downloads single video
- `main_playlist.py` — Downloads YouTube playlist
- `main_loop_playlist.py` — Batch loop mode for continuous downloads

### yt-dlp Abstraction
- yt-dlp handles YouTube API interaction and video downloading
- curl_cffi for HTTP requests with fingerprint customization
- FFmpeg for post-processing and format conversion

### Simple Pipeline Pattern
```
User Input → Parse Arguments → yt-dlp Download → FFmpeg (optional) → Output File
```

## Key Dependencies

## Architecture Decisions

1. **yt-dlp over youtube-dl** — Active fork with more features and better maintenance
2. **curl_cffi** — TLS fingerprint customization for avoiding detection
3. **Multiple entry scripts** — Separate scripts for single, playlist, and loop modes
4. **No web deployment** — Pure CLI tool for local usage
5. **FFmpeg dependency** — Required for post-processing and format conversion
