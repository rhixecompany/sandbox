### 1. Seed API Route — `src/app/api/seed/route.ts`

A well-structured API route handler with Zod validation, server action pattern, and consistent response format.

**Pattern:** Server Action + Zod Validation  
**Category:** API Design

```typescript
"use server";
import { type NextRequest, NextResponse } from "next/server";
import { z } from "zod";

interface SeedApiResponse {
  code?: number;
  data?: unknown;
  error?: string;
  message?: string;
  ok: boolean;
}

const seedRequestSchema = z.object({
  entities: z
    .union([
      z.enum([
        "types",
        "authors",
        "artists",
        "genres",
        "comics",
        "chapters",
        "users",
        "all"
      ]),
      z.array(
        z.enum([
          "types",
          "authors",
          "artists",
          "genres",
          "comics",
          "chapters",
          "users",
          "all"
        ])
      )
    ])
    .optional(),
  options: z
    .object({
      batchSize: z.number().positive().optional(),
      concurrency: z.number().positive().optional(),
      verbose: z.boolean().optional(),
      dryRun: z.boolean().optional(),
      forceOverwrite: z.boolean().optional()
    })
    .optional()
});
```

**Why it's exemplary:** Consistent response interface, type-safe Zod schema with union types, batched/concurrent operation options.
