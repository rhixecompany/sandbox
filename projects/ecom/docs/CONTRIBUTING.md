# Contributing to RhixeCompany Ecom

## Introduction

Thank you for your interest in contributing to RhixeCompany Ecom! This document provides guidelines for contributing to the project.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally: `git clone https://github.com/YOUR_USERNAME/ecom.git`
3. **Create** a feature branch: `git checkout -b feature/your-feature`

## Development Workflow

### Setting Up Development Environment

See [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md) for detailed setup instructions.

### Making Changes

1. Make your changes in your feature branch
2. Test your changes locally
3. Commit your changes with descriptive commit messages
4. Push to your fork
5. Create a pull request

## Code Standards

### Python (Django Backend)

- Follow PEP 8 style guide
- Use meaningful variable and function names
- Add docstrings to classes and functions
- Keep functions focused and modular

### JavaScript (React Frontend)

- Use functional components with hooks
- Prefer `const` over `let`
- Use meaningful names
- Add comments for complex logic

### Git Commit Messages

Use clear, descriptive commit messages:
- `Add user authentication feature`
- `Fix cart total calculation bug`
- `Update product listing pagination`

Avoid:
- `Fix stuff`
- `WIP`
- Unclear or vague messages

## Pull Request Guidelines

### Before Submitting

1. **Test your changes** - Ensure code works correctly
2. **Check formatting** - Run linting tools if available
3. **Update documentation** - Reflect any API changes in docs
4. **Review your code** - Check for errors and improvements

### Pull Request Description

Include:
- Brief description of changes
- Related issue number (if applicable)
- Testing performed
- Screenshots (for UI changes)

### What Happens Next

1. Code review by maintainers
2. Feedback may be provided
3. Request changes if needed
4. Merge when approved

## Types of Contributions

### Bug Reports

When reporting bugs:
- Describe the issue clearly
- Include steps to reproduce
- Provide environment details
- Include screenshots if relevant

### Feature Requests

When requesting features:
- Describe the feature
- Explain the use case
- Consider implementation approach
- Provide mockups if available

### Documentation

Help improve:
- README files
- Code comments
- User guides
- API documentation

### Code Contributions

Areas where contributions are welcome:
- Bug fixes
- New features
- Performance improvements
- Test coverage
- Documentation

## Questions?

If you have questions:
- Open an issue for discussion
- Contact the maintainers directly

## Recognition

Contributors will be recognized in:
- Project documentation
- Release notes
- GitHub contributors list

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow
- Focus on what is best for the project
