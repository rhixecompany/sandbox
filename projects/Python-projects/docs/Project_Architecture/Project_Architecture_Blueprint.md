## Architectural Patterns

### Standalone Script Pattern
Each script is independently executable with:
- Its own entry point (`if __name__ == "__main__"`)
- No shared module structure
- Direct dependencies from `requirements.txt`

### Utility Collection Pattern
Scripts are loosely organized by function:
- **Calculators:** Basic math operations
- **Games:** Interactive command-line games
- **Graphics:** Image processing and visualization
- **Web:** Scraping and API interactions
- **Utilities:** Automation and scheduling

1. **Standalone scripts** — No shared framework; each script is self-contained
2. **No formal testing** — Scripts are educational/experimental
3. **PEP 8 compliance** — Python community coding standards
4. **Simple dependency management** — Single `requirements.txt`
