## Workflows

### Workflow 1: Blog Post Creation & Publishing

**Description:** Admin user creates, edits, and publishes blog posts with CKEditor rich text editing.

**Entry Points:**
- Django admin interface
- Custom views in `base/views.py`

**Steps:**
1. Author logs into Django admin or custom post editor
2. Author composes blog post using CKEditor 5 WYSIWYG editor
3. Author sets metadata: slug, thumbnail, categories, publish date
4. Author saves as draft or publishes immediately
5. On save: post written to PostgreSQL via Django ORM
6. Thumbnail images uploaded to Google Cloud Storage
7. Signal generates post slug from title
8. Published post appears on blog listing page
9. Visitors can view post and leave comments
10. Comments are moderated via admin

**Error Handling:**
- GCS upload failure → fallback to local storage, retry enqueued
- Missing slug → auto-generated from title in `signals.py`
- CKEditor XSS → Django template escaping + bleach sanitization

**Test Patterns:**
- `python manage.py test` — `base/tests.py`
- URL resolution tests for all views
- Model validation tests

### Workflow 2: Portfolio Project Showcase

**Description:** Visitor browses portfolio projects with filtering and search.

**Entry Points:**
- Public portfolio pages in `base/views.py` (CBVs)

**Steps:**
1. Visitor navigates to portfolio section
2. Class-Based View queries projects from database
3. Projects are rendered with thumbnails, descriptions, tech stack tags
4. Visitor can filter by category or search by keyword
5. Clicking a project shows full detail page
6. Project images served from Google Cloud Storage

**Error Handling:**
- Missing project → 404 page
- GCS image missing → fallback placeholder
