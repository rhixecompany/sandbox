## Architectural Patterns

### Django Full-Stack Pattern
- Django handles both backend and frontend rendering
- Class-Based Views (CBVs) preferred for consistency
- Template-based rendering with Django template language

### Cloud Storage Pattern
- Google Cloud Storage for media file hosting
- Static files served via GCS
- Environment-based configuration for storage backends

### Rich Content Pattern
- CKEditor 5 for WYSIWYG content editing
- Markdown support in blog posts
- Draft and publish workflow for content

## Architecture Decisions

1. **Class-Based Views** — Consistent CBV pattern across all views
2. **GCS for media** — Cloud-based file storage for scalability
3. **CKEditor 5** — Modern rich text editing with image handling
4. **GCP deployment** — Google Cloud Platform for hosting and storage
5. **Docker containerization** — Consistent deployment across environments
