# Archived: Skills Commit Batch Scripts

**Date archived:** 2026-05-31
**Reason:** One-shot migration artifacts from skills-commit migration project

## What these are

These scripts were auto-generated to commit skills files in batches of ~50.
They were created by a batch-processing tool and have no current references
from package.json, Makefile, or any active script.

## Contents

- 26 `.sh` variants (skills-commit-batch-1.sh through skills-commit-batch-26.sh)
- 25 `.ps1` variants (skills-commit-batch-1.ps1 through skills-commit-batch-25.ps1)

Each script has the same structure — only the file paths differ.
They were used exactly once to commit the skills library migration
and have no ongoing purpose.

## Why archive instead of delete

Per the bash-scripts-audit-remediation skill's archival pattern:
one-shot migration artifacts should be archived (moved) rather than deleted
in case historical reference is needed. They are not active code.
