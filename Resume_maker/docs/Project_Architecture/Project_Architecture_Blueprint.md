## Architectural Patterns

### Pipeline Processing Pattern
The generator follows a sequential pipeline:

1. **Input**: Read structured JSON data (personal info, experience, education)
2. **Transform**: Process data into formatted content
3. **Render**: Generate Markdown output with templates
4. **Export**: Produce PDF via markdown-pdf conversion

### Single-Entry Architecture
`index.ts` is the single entry point that handles:
- Command-line argument parsing (`--input`, `--output`, `--format`)
- Data loading from JSON input files
- Document generation pipeline
- Multi-format output (Markdown + PDF)

## Key Dependencies

## Architecture Decisions

1. **Bun-first** — Uses Bun for execution, package management, and TypeScript transpilation
2. **Single entry point** — `index.ts` handles all document types via a unified interface
3. **JSON input** — Structured data input via JSON files for reproducibility
4. **Dual output** — Markdown for editing/sharing, PDF for final delivery
5. **Sample-first** — `sample-input.json` serves as template and test fixture

## Extensibility Points

- Add new document types (e.g., portfolio, bio) in the pipeline
- Add new output formats (e.g., DOCX, HTML)
- Support additional input formats (YAML, TOML)
- Add template customization via command-line options
